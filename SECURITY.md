# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch.

## Reporting a Vulnerability

Please do not file public issues for security vulnerabilities.

Instead:
1. Open a GitHub Security Advisory draft for this repository, or
2. Contact the maintainer directly with:
   - reproduction steps
   - impact assessment
   - suggested mitigation (if available)

You can expect an acknowledgment within 72 hours.

## Sensitive Data

Never include personal credentials, tokens, or private logs in bug reports or pull requests.
